# Evidence logs

These append-only logs preserve the reasoning and commands that are too operational for design documents but too important to lose.

| Log | Records |
| --- | --- |
| [build.md](build.md) | Meaningful implementation milestones and affected contracts. |
| [experiments.md](experiments.md) | Compiler counterfactuals, commands, inputs, and artifacts. |
| [validation.md](validation.md) | Validation commands, versions, results, and coverage. |
| [failures.md](failures.md) | Failed approaches, root causes, and resulting safeguards. |
| [model-usage.md](model-usage.md) | Material GPT-5.6/Codex decisions, inputs, and verification. |
| [benchmarks.md](benchmarks.md) | Raw-run references, environment, statistics, and rejected measurements. |
| [security.md](security.md) | Threat-model reviews, policy regressions, and disclosure-safe fixes. |
| [releases.md](releases.md) | Release contents, compatibility, and retained evidence. |

Use UTC ISO 8601 timestamps. Append corrections rather than rewriting history. Never record secrets, private source, access tokens, or raw environment dumps.
